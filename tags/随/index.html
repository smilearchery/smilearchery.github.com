实习三个月有感

从六月份开始在公司实习到现在已经三个月有余了，总觉得应该写点什么，文笔不佳，权当是做些记录吧~

一开始进入公司配置开发环境，熟悉工作流程用了三四天。然后开始解bug，解bug这段时间主要熟悉了如何使用git(我们版本管理用的gerrit)，如何在Android源码环境下进行开发/编译等。这段时间也算是leader对我的一个考察阶段，看看我对Android的熟练程度，还好我对Android应用开发的基础还好，总得来说刚入职的几天还算顺利。

然后开始单独做小应用，先是完成了一个apk的检验工具，这个以后会写文讲具体的实现过程，然后是做leader的一个小项目。没几天卫士上马，开始加入卫士项目，一期从八月中到九月初，二期从九月中至十一，这算是我第一次完整的经历了一个产品的诞生及完善的过程。

首先是产品论证阶段，前期论证不清楚，后期论证大概用了一周左右的时间，几乎工具组的所有同学都加入了讨论，论证的内容包括与现有产品的关系，与近品的比较，加入什么功能，用户群，潜在的用户数量等等方面…论证之后老板说可以开搞之后进入产品设计阶段。

产品设计阶段主要是项目内的人员，包括Coder,UI,产品经理。产品经理会先设计好大概功能，各页面的大概交互，使用逻辑等，然后和大家一起讨论。顺便多一嘴，讨论是个神奇的东西，不仅能发现问题，也能激发创意，而且我觉得如果能把公司的阿姨邀请进来一起讨论会有更好的效果。

设计阶段过后就是编码阶段了，每个人会有不同的分工和排期，即在规定的时间内完成规定的任务。其中我负责UI层的编写，底层的接口由另外几个老家伙提供。

编码完成后会有测试及show，show的过程一般工具组的同学都会参加，大家来找茬，有问题会记录下来解决或加入下一期。最后没有问题就可以走发布流程。

这里稍微写的细一点(当然没有特别详细，详情可见 《软件工程》)是因为这是第一次完整的经历一个产品的诞生过程，真正的了解了如何从一个idea到产品的过程，这个经历是无比珍贵的，也可以算是我实习以来最大的收获。

找实习的那点事儿

从去年七月份开始漫不经心的准备实习，到现在已将近一年，总算拿到了两份Offer，也算验证了自己这段时间的准备没有白费(当然也没好到哪去，要不也不会这么费劲，也不会浪费很多好的机会)，于是想着总结下这段时间的经验和教训，一方面督促下自己，一方面为下学期开始的校招做好准备.
算一算，到现在参加的笔试有三次，分别是：

  1. 11年新浪网络学院，Android开发方向，无情的被刷了；
  2. 今年5月腾讯实习生招聘，Android开发方向，RP爆表通过笔试；
  3. 今年5月百度实习生招聘，C++开发方向，笔试后无消息(好吧，我承认百度笔试我基本没答对几道题)。

面试有5次：

  1. 今年5月腾讯实习生招聘通过笔试后一面，然后就没然后了；
  2. 今年6月某外包公司两面(C#方向)，表示让熟练C/Java的人面C#的结果就是被完爆；
  3. 今年6月某创业公司一面，拿到Offer；
  4. 今年6月创新工场点心OS电话一面，拿到Offer；

从目前经历过的来看，各公司的招人的风格都不太一样，比较大的公司如sina,tx,baidu一般会安排大规模的笔试和面试(除了内推，一般可以免笔试，直接面试)。比较小的公司一般可能不会有笔试环节，而是直接进行面试。个人认为，一般来说面试的变数较大，和自身的实力虽然有些关系(大牛的话就表提了)，但是和面试官的个人喜好也有很大的关系，有很大的运气成分在里面。而笔试比较固定，一般会和各公司风格有关，不会有太大变化。

先说一下笔试，不管是前端还是后端，基础很重要。笔试一般跟各公司风格有关，拿我参加过的笔试来说，新浪和腾讯考得东西非常多但是非常基础，基本上逃不了计算机网络，操作系统，数据结构，算法，数据库等基础知识(总之就是基本上所有计算机的基础课，咩哈哈~)，腾讯是不管前端后端都是基本一套题(个别有差异)，新浪网络学院的考试则多了一个大约300字左右的计算机专业英语短文翻译和两道智力题。而百度则是十多道算法大题，要求手写代码或写出解决方案，非常高端(非常无奈有木有T-T)，只有一道C/C++的语言基础题。

面试的话，一般是四分技术，四分基础，两分RP(当然大牛的话就没这么多事儿了，并且如果算法好的话可以无视其他方面，一俊遮百丑啊)。一般面试的流程基本上开始是先问一些基础问题探探底，如语法啦，网络啊什么的，然后根据面试官的喜好不同可能会根据你做过的东西问一些问题，比如你在项目中扮演的角色，解决了什么问题，用了什么技术等等。也有可能问你些算法问题，当然更有可能的是两者都问一点(像我这种算法不好的只有祈求上苍了==)。比如说腾讯面试的时候是一对一面试，面试官一般看着我的简历，一般玩电脑，问了我从C的语法基础(static,define等)，到数据结构，到项目，还让当场手写链表逆置的算法(没写完整，只把思路写出来了，手写代码无感觉的说，估计就挂在这上了)。后三个公司因为有两个是电话面试，所以基本上问题集中在基础知识，项目及技术实践上。

Ok~以上算是我经历的笔试及面试的一点经验，下面讲一下如何学习及准备。

计算机不是一个纯理论的学科，除非你去研究算法啊加密啊这种非常理论化的东西，否则基本是离不开动手的，所以不要以为能从书里学习到所有的东西(当然书本也是必不可少的)。看完了书，然后动手去实践是最好的学习方法。

同时，计算机可以说是技术更新最频繁的学科，新技术层出不穷。每一门技术都有可能独霸天下，也有可能昙花一现。比如说谁能想到曾经不可一世的Symbian会衰落至此，谁又能想到有巨头撑腰的Meego最后却昙花一现?如果说别的行业给了你太少的选择，那么计算机绝对会让你在选择方向时眼花缭乱。在选择成熟的方向会比较安全，但是因为成熟，所以竞争压力也大，你只有有了相当的深度才能立足；而选择新的技术则需要眼光，一个好的技术可以让你迅速的在这个行业有着一席之地，当然也有着相当的风险，比如说谁能想到去年Android开发方向会如何火爆，而今年却市场冷清。如果喜欢新技术，个人建议不妨先选择一个成熟的技术以立足，这样在研究新技术时心里会有底些～

当然也有童鞋(包括我)，总是认为一些课(比如高数啊电路啊等等)没有用，技术(更准确的说应该称为技能)才是王道，其实技术才是次要的，一门新的技术如果你的基本功好的话可以在短时间内掌握，而如果基本功不扎实就很难补了。一个人的基本功对未来的职业生涯有着相当大的影响，比如说电路，先不说你个理工科电力大学出来不会搞电是不是太水(我就水的一塌糊涂==)，IT有个重要的方向就是嵌入式，其中涉及到很多电路数电模电的知识，这些学不好以后想干这个基本上很困难。还有其他的我就不一一举例了，总之学好每一门课(特别是计算机的基础课如操作系统，网络，数据结构等等)是很重要的。从各公司招聘的笔试面试也可以看出来，他们对基础的要求是相当高的。

其实对各位的建议是最好能多研究算法，而且越深越好，从目前的招聘来看，基本上算法好可以某种程度上无视其他条件如项目经验等，如果能在比赛(如ACM,有道等)中获奖会有很大的加分，而且未来发展也会比专攻某一技术的好很多。而且各大公司招聘也对算法有一定要求，有事没事研究一下没坏处的。同时，各种竞赛也可以参加一下，像什么挑战杯，微软创新杯，智能车，Google应用挑战等等等等。有了这些经验也会为你的简历增色不少。而且有一个成型的作品也会让你有更多的机会。

最后讲一些技巧性的东西，我只说概念，以后有心情了写详细点，但是各位还是多多google baidu吧~

修改密码攻略之数据结构

最近CSDN等网站泄漏用户信息的事件闹得沸沸扬扬，网传多个网站包括人人网，猫扑等的用户信息都已遭泄露。虽然个别网站的真实性仍有待考证，但是我们最好加强防范，尽快更改相关密码，有备无患才是。 好，下面进入正题，我们如何设置复杂好记难以攻破的密码呢?
首先，我们需要了解一下几种最常用的密码破解的原理：

  1. 暴力穷举 密码破解技术中最基本的就是暴力破解，也叫密码穷举。如果黑客事先知道了账户号码，如邮件帐号、QQ用户帐号、网上银行账号等，而用户的密码又设置的十分简单，比如用简单的数字组合，黑客使用暴力破解工具很快就可以破解出密码来。因此用户要尽量将密码设置的复杂一些。
  2. 木马 如果用户密码较为复杂，那么就难以使用暴力穷举的方式破解，这时黑客往往通过给用户安装木马病毒，通过记录键盘等方法获取用户密码
  3. 不良习惯 有一些公司的员工虽然设置了很长的密码，但是却将密码写在纸上，还有人使用自己的名字或者自己生日做密码，还有些人使用常用的单词做密码，这些不良的习惯将导致密码极易被破解。
  4. 密码心理学（社会工程学） 很多著名的黑客破解密码并非用的什么尖端的技术，而只是用到了密码心理学，从用户的心理入手，从细微入手分析用户的信息，分析用户的心理，从而更快的破解出密码。其实，获得信息还有很多途径的，密码心理学如果掌握的好，可以非常快速破解获得用户信息。

其中第一点和第四点可以归为一类，因为黑客在破解密码的过程中并非单纯的使用组合，而是要经过一定的筛选，对受害人进行一定的分析并制作相应的字典(破解时用于穷举)，这样成功率才会比较高，时间也相应比较快。
